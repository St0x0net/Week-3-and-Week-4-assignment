# Week3-and-Week-4-Practice-Problems
## Overview
This codebase implements fundamental Sorting and Searching algorithms applied to financial scenarios such as transaction auditing, risk management, and market trend analysis.

## Modules Included
### Trade Volume Analysis (Merge Sort): Handles large-scale market data. Uses a stable $O(n \log n)$ approach to sort volumes without losing the relative order of identical values.
### Portfolio Rebalancing (Quick Sort): Optimized for speed. Uses an in-place $O(n \log n)$ average-time complexity algorithm to organize asset prices.
### Transaction Forensics (Binary Search): A high-speed search utility for sorted logs. Reduces search time from $O(n)$ to $O(\log n)$, essential for auditing millions of records.
### Risk Threshold Lookup: Identifies boundaries (Floor/Ceiling) within risk bands to categorize clients and assign compliance levels dynamically.

## Key Concepts
### Stability: Preserved in Problem 3 to ensure audit trails remain intact.

### Efficiency: Problems 3 and 4 move beyond O(n^2) "simple" sorts to O(nlogn) "complex" sorts for professional-grade performance.

### Search Optimization: Prerequisites for Problems 5 and 6 require data to be pre-sorted to achieve logarithmic search speeds.
